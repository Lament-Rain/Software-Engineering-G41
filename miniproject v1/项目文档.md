# BUPT 国际学校 TA 招聘系统项目文档

## 1. 项目概述

本项目是为 BUPT 国际学校设计的 TA 招聘系统，旨在简化 TA 招募流程，提高招聘效率，实现公平、透明的招聘过程。系统覆盖 TA、MO、管理员三个核心角色，提供完整的招聘全流程管理功能。

## 2. 系统架构

### 2.1 目录结构

```
src/
├── model/          # 数据模型层
├── service/        # 业务逻辑层
├── controller/     # 控制层（命令行界面）
├── utils/          # 工具类
├── data/           # 数据存储目录
└── Main.java       # 主程序入口
```

### 2.2 核心模块

| 模块名称 | 主要功能 | 文件位置 |
|---------|---------|----------|
| 用户管理 | 账号注册、登录、档案管理 | src/service/UserService.java |
| 职位管理 | 职位发布、编辑、终止 | src/service/JobService.java |
| 申请管理 | 职位申请、状态管理 | src/service/ApplicationService.java |
| AI 智能 | 技能匹配、工作量平衡 | src/service/AIService.java |
| 公共功能 | 数据导出、系统搜索 | src/service/CommonService.java |
| 数据存储 | 数据持久化、文件操作 | src/service/DataStorage.java |

## 4. 技术实现要点

### 4.1 数据存储

- 使用文本文件存储数据，位于 `src/data/` 目录
- 主要存储文件：
  - `users.txt`：用户信息
  - `jobs.txt`：职位信息
  - `applications.txt`：申请信息
  - `logs.txt`：系统日志

### 4.2 安全措施

- 密码加密存储
- 账号锁定机制（连续 5 次错误登录）
- 操作日志记录

### 4.3 核心功能实现

- **用户注册**：支持手机号/邮箱注册，验证身份归属
- **职位发布**：MO 创建并发布 TA 招募职位，支持草稿保存
- **申请管理**：TA 申请职位，MO 筛选和录用
- **AI 匹配**：基于技能和时间的智能匹配
- **工作量平衡**：监控 TA 工作量，避免过度分配

## 5. 系统使用说明

### 5.1 登录方式

- 系统默认管理员账号：admin1 / admin123
- TA 和 MO 账号需要注册后登录

### 5.2 主要操作流程

1. **TA 流程**：注册账号 → 完善个人档案 → 浏览职位 → 申请职位 → 查看申请状态
2. **MO 流程**：注册账号 → 发布职位 → 查看申请 → 筛选申请人 → 录用 TA
3. **管理员流程**：登录系统 → 审核账号/档案/职位 → 监控申请流程 → 管理工作量 → 配置系统参数

## 8. 注意事项

1. 所有代码必须遵循 Java 编码规范
2. 模块间通过服务类进行通信，避免直接操作数据
3. 所有操作必须记录日志，便于问题排查
4. 数据存储使用相对路径，确保跨平台兼容性
5. 定期备份数据，防止数据丢失
